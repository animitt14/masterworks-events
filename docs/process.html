RSVP Dashboard Process

1 · Pull RSVPs

1
Go to Redash query #11069. Set the Date parameter to the event date (YYYY-MM-DD) and run the query.
2
Download the results as a CSV.

2 · Import to HubSpot

1
In HubSpot, go to Contacts → Import. Upload the CSV.
2
Map columns to HubSpot properties. Where prompted, check "Don't override existing values" to avoid overwriting any data already in HubSpot.

3 · Enrichment

After import, contacts often have no job title or company. There are two enrichment layers:

Auto 8am / 12pm / 4pm daily rebuild

Runs via GitHub Actions (daily.yml). Fetches RSVPs for today and upcoming events, searches LinkedIn and other public sources for blank title/company (up to 95 calls/day), looks up NYC property values via PLUTO, enriches photos via RocketReach, fills company from work email domain as a fallback, then rebuilds and publishes index.html.

Auto 8:30am enrichment routine

A separate scheduled agent runs after the rebuild. It web-searches today's RSVP contacts still missing title/company (LinkedIn, ZoomInfo, company pages), patches HubSpot directly, and does a second pass on anyone still blank.

4 · Scoring

Every contact gets a score of 1–5 computed automatically from title, company, email domain, estimated net worth, and lifecycle stage.

  • 5 (High) — Finance domain, physician, C-suite at named top-tier firm, already invested or Opportunity stage
  • 4 (Medium-High) — VP / Director level, finance firm without top-tier signal
  • 3 (Medium) — Senior manager, attorney, mid-tier signals
  • 2 (Low-Medium) — Founders (unverified), analysts, no-shows, no data
  • 1 (Low) — Wealth advisors (refer, don't invest), art world, real estate agents, not interested

To override a score for a specific contact, edit SCORE_OVERRIDES in generate_rsvp.py — add the HubSpot contact ID and the desired score (1–5). Changes go live on the next rebuild.

5 · Uninviting & Confirmations

These events are curated — only contacts who can invest $25k or more should be in the room. Use the scoring and enrichment data to identify anyone who doesn't meet that bar and uninvite them before the event.

Uninviting: The dashboard has an uninvite checkbox per contact. Checking it writes to a shared Gist. The next rebuild syncs those to HubSpot (sets outbound_event_attendee_disqualified) and clears the Gist. A HubSpot workflow listens for that property change and automatically sends the uninvite email to the contact.

Send confirmation: The dashboard also has a send-confirmation checkbox. Checking it sets outbound_event_send_confirmation on the contact. A HubSpot workflow listens for that property and automatically sends the confirmation email.

First-time setup: The first time you check either box, the dashboard will prompt for a GitHub personal access token. This is required for security so changes can be written to the shared Gist. Ask Anisha for the team token — it saves in your browser and you won't be asked again.

6 · Editing the Code

The dashboard is generated by generate_rsvp.py in the animitt14/masterworks-events repo. To make a change: edit generate_rsvp.py, commit, and push to main — GitHub Actions rebuilds the dashboard within a few minutes.

Never edit docs/index.html directly — it is overwritten on every rebuild.

Pushing without conflicts

GitHub Actions commits to main automatically several times a day. If you try to push and get a rejection, do not use git reset --hard — that will destroy your local changes. Instead run:

  1. Save your edited files
  2. git fetch origin
  3. git reset --soft origin/main — syncs with remote but keeps your edits staged
  4. git add generate_rsvp.py (or whichever files you changed)
  5. git commit -m "describe your change"
  6. git push

For anything more involved, open Claude Code in the repo directory and describe the change — it knows the full codebase.

Automated schedules: full rebuild at 8am, 12pm, and 4pm ET via GitHub Actions; enrichment agent at 8:30am ET via Claude Code. Manage scheduled routines at claude.ai/code/routines.

7 · Dependencies

These accounts and credentials keep the system running. If ownership needs to transfer, each item below needs to be handed off.

  • GitHub repo — animitt14/masterworks-events. Hosts the code and serves the dashboard via GitHub Pages. GitHub Actions secrets (HubSpot key, Google key, RocketReach key) are stored under this repo.
  • GitHub Gist — uninvite sync uses a private Gist under the same account. New owner would need to create a replacement and update the Gist ID in generate_rsvp.py.
  • Claude Code account — the 8:30am enrichment routine runs as a scheduled agent tied to the account that created it. Manage or recreate at claude.ai/code/routines.
  • HubSpot API key — stored as a GitHub Actions secret and in the local .env file (gitignored). Tied to a HubSpot user account; rotate if that account changes.
  • Google Custom Search API key — used for enrichment (~95 searches/day quota). Stored in GitHub Actions secrets and .env.
  • RocketReach API key — used for contact photo enrichment. Stored in GitHub Actions secrets and .env.
